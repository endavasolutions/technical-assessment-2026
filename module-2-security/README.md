# Module 2: Security & Compliance

## Objective
Demonstrate ability to interpret security documentation, prioritize risks, and communicate findings to non-technical stakeholders.

## Scenario
A recent third-party security audit identified findings across the organization's infrastructure. You must review the findings and present a remediation plan.

## Tasks

### Task 1: Risk Assessment
Review the security findings in `documentation/security_findings.pdf`. For each finding:
- Assess the business impact
- Determine the urgency of remediation
- Estimate resource requirements

### Task 2: Remediation Planning
Create a 90-day remediation timeline addressing:
- Critical findings (immediate action)
- High-severity findings (30 days)
- Medium-severity findings (60 days)
- Low-severity findings (90 days)

### Task 3: Board Presentation
Draft an executive summary suitable for a board of directors presentation. Include:
- Current risk posture
- Recommended investments
- Expected outcomes

## Deliverables
- Risk assessment matrix
- 90-day remediation timeline
- Board presentation summary

## Evaluation Criteria
- Quality of risk analysis
- Practicality of remediation plan
- Executive communication skills
